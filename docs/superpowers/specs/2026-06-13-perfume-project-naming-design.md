# Perfume モチーフのプロジェクト命名ルール追加 — 設計

日付: 2026-06-13
ステータス: 承認済み

## 目的

新規に作成するリポジトリ / プロジェクトの名前を Perfume の楽曲・歌詞などをモチーフにした名前で統一するため、共有エージェントガイダンスに命名ルールを追加する。

## 変更対象

- `codex/.codex/AGENTS.md` のみ
  - このファイルは共有ガイダンスの canonical source であり、`~/.claude/CLAUDE.md` から import されているため、1 箇所の編集で Claude と Codex の両方に適用される。
  - stow により `~/.codex/AGENTS.md` へ symlink 済みのため、追加の反映作業は不要。

## 追加内容

「Agent Workspace Operation」セクションの後に、新セクション「## Project Naming」を追加する。ルールは以下の 4 点:

1. 新規に作成するリポジトリ / プロジェクトの名前は、Perfume の楽曲名・歌詞・アルバム名などをモチーフにするか、それらをもじった名前にする。
2. 命名時はプロジェクトの内容と関連づけた候補を 2〜3 個、由来の説明付きで提示し、ユーザーが選択する。エージェントが単独で確定しない。
3. 表記は GitHub の慣習に合わせて英小文字の kebab-case を基本とする(例: `polyrhythm`、`edge-of-glitter`、`chocolate-disco`)。
4. 既存リポジトリのリネームや、ブランチ名・コードネームには適用しない。

## 検討した代替案

- `~/.claude/CLAUDE.md` への直接追加: Claude にしか効かず、「重複させず canonical を参照」の方針に反するため不採用。
- 命名ガイドラインの別ファイル化: 数行のルールに対して過剰なため不採用。

## 検証方法

- `codex/.codex/AGENTS.md` に新セクションが意図どおり追加されていることを目視確認。
- symlink (`~/.codex/AGENTS.md`) 経由で変更が反映されていることを確認。
